About me

I am a proactive financial sector professional with a strong interest towards data science. In my day-to-day work I validate credit risk models using mainly Python. This website is my personal initiative to refamiliarize myself with the possibilities of modern website development.

I still remember when me and my friends used to learn to code in the beginning of the 21st century writing raw HTML code and using sophisticated tools like notepad and paint. I regarded it highly intriguing although the website creation was usually deemed of not having any sensible content to put on the site.

During the recent years, it has been fun to observe the big hype around the website and app creation as the concepts have, as seen through my still limited understanding, remained quite similar to the ones I learned 15 years ago. In this website, I've written mainly HTML together with an associated CSS file so I've concentrated on the front-end part. I hope to get a chance to familiarize myself with the vast amount of back-end technologies also in the future.

  • Background: M.Soc.Sc. (Econ.)
  • Field: Financial services
  • Interests: Modeling, Data science

Skills

Python

My newest old love is Python. I've been using Python throughout my studies for the basic data analysis and visualization tasks as it enjoys a solid appreciation among the data science community.

Lately, I've picked it up again to truly get a grasp of the versatility of its modern libraries.

R

When I changed my major to Economics, I quickly realized R is the go-to tool in statistical and economic data analysis. I have been using R daily for around five years now including writing almost all of my client project deliverables in it.

I have seen the effect of this by slowly transferring from questions like "How can I write this in R" to "How can I write this in an clean and understandable manner in R".

SQL

Another interesting addition to my competences has been SQL as I quickly found out that the ability to make quick and efficient queries is highly useful in the professional setting. Almost all clients I've worked with have a some sort of SQL-based solution housing the vast amound of data they possess.

I've picked up the SQL on-the-go as the real world data collection usually differs a lot from the university courses' pre-cleaned datasets.

LaTeX and Markdown

Latex is one of my older loved-ones. I was mezmerized by its simplicity and customizability while studying Physics back in the day. Afted adopting the syntax, I've used it in writing my CV, Master's thesis and a number of other research reports.

Markdown is a newer acquaintance that I learned while writing code-heavy problem set solutions in Econometrics. After that, I've written most of the client projects' model documentation using Markdown.

HTML & CSS

This website is a good example of my HTML and CSS competences. I wanted to make it exactly like I imagined and not just settle for some pre-made template available on the web.

Next, I try to continue adding functionalities and content here as a part of the routine of updating my GitHub.

Git & GitHub

I've just recently found out about the awesomeness of Git. Many times I've said to myself "There has to be a better way" when reviewing the colorful practices for version control of different companies.

Until now, I've used Git only for my personal projects, not in a professional setting, hence the low commit frequency. The more I am getting familiar with Git, the more I long to use it in my work also.
